# Требования к инфраструктуре

**Aviation Fuel Management System**  
Версия документа: 1.0 | Май 2026

---

## Аппаратные требования

### Минимальная конфигурация
*(до 10 одновременных пользователей, ~1 000 операций/день)*

| Параметр | Значение |
|----------|----------|
| CPU | 2 ядра (x86_64 / ARM64) |
| RAM | 4 ГБ |
| Диск (SSD) | 50 ГБ |
| Сеть | 10 Мбит/с |
| ОС | Ubuntu 22.04 LTS / Debian 12 |

### Рекомендуемая конфигурация
*(10–50 одновременных пользователей, ~10 000 операций/день)*

| Параметр | Значение |
|----------|----------|
| CPU | 4 ядра |
| RAM | 8 ГБ |
| Диск (NVMe SSD) | 100 ГБ |
| Сеть | 100 Мбит/с |
| ОС | Ubuntu 22.04 LTS |
| Резервирование | RAID-1 или snapshot |

### Оценка роста диска

| Сценарий | БД/год | Бэкапы/год |
|----------|--------|------------|
| 500 операций/день | ~2 ГБ | ~10 ГБ |
| 2 000 операций/день | ~8 ГБ | ~40 ГБ |
| 10 000 операций/день | ~40 ГБ | ~200 ГБ |

*(Включая audit_log — самую быстрорастущую таблицу)*

## Программные требования

| ПО | Минимальная версия | Назначение |
|----|-------------------|-----------|
| Docker Engine | 24.0 | Контейнеризация |
| Docker Compose plugin | 2.20 | Оркестрация |
| Linux Kernel | 5.4+ | Поддержка cgroups v2 |
| openssl | 1.1+ | Генерация секретов |
| Git | любая | Получение обновлений |

### Опциональное ПО

| ПО | Назначение |
|----|-----------|
| Nginx | Reverse proxy, HTTPS termination |
| certbot | SSL-сертификаты Let's Encrypt |
| rsync | Синхронизация бэкапов на удалённый сервер |
| AWS CLI / MinIO Client | Загрузка бэкапов в объектное хранилище |

## Сетевые требования

| Порт | Протокол | Направление | Назначение |
|------|----------|-------------|-----------|
| 80 | TCP | Входящий | HTTP (редирект на HTTPS) |
| 443 | TCP | Входящий | HTTPS (основной доступ) |
| 5000 | TCP | Входящий | Прямой доступ к приложению (если без nginx) |
| 5432 | TCP | Внутренний | PostgreSQL (только внутри Docker-сети) |

**PostgreSQL (5432) должен быть закрыт для внешнего доступа.**
