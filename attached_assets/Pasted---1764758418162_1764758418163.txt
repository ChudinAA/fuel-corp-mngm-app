Необходимо подкорректировать для Справочников логику, схему БД, и саму БД:

Сущность "Покупатель" должна быть одна для "Новая запись: ОПТ" и "Новая запись: Заправка ВС", то есть покупатели должны сохраниятья в одну отдельную таблицу БД, и при дальнейших флоу выбора покупатлей на других экранах, они должны браться из таблицы Покупателей. То есть на странице "Справочники" свехру где переключатель разделов "ОПТ"/"Заправка ВС"/"Логистика"надо добавить еще пункт "Покупатели" и по аналогии с текущими страницами сделать возможность добавить новую запись.
Когда открываю окно "Новая запись: ОПТ" для создания новой записи в справочнике, на UI на первом месте должен быть выпаюащий списко "Тип", а ниже "Название" (поменять очередность этих полей), далее - если я выбираю тип Поставщик, то ниже должно добавляться выпадающее поле "Базис поставки", связанное с этим поставщиком, соответствено надо добавить новую таблицу ("Базисы поставки") в БД для этой связи/логики, и будут добавляться в эту таблцу базисы, когда я буду создавать новую запись с типом Базис поставки. Необходимо корректно сделать измнения в схеме БД и в самой БД. В выпадющем списке "Тип" должно быть два варианта: "Поставщик", "Базис поставки" (Покупатель теперь как отдельная вкладка страницы).
Для окна "Новая запись: Заправка ВС" - аналогично пункту выше, надо поменять поля "Тип" и "Название" местами, а также при выборе типа "Аэропорт" (переимновать в "Аэропорт/Поставщик") ниже должно добавляться выпадающее поле "Базис заправки", связанное с этим поставщиком, соответствено надо добавить новую таблицу ("Базисы заправки") в БД для этой связи/логики, и будут добавляться в эту таблцу базисы, когда я буду создавать новую запись с типом Базис заправки. В выпадющем списке "Тип" должно быть два варианта: "Аэропорт/Поставщик", "Базис заправки". Также на окне "Новая запись: Заправка ВС" на UI нет опциональных полей "Стоимость услуги", "Стоимость ПВКЖ", "Агентские/прочие" (хотя в таблицу в БД есть поля для этого - надо привязать).
Для окна "Новая запись: Логистика" также поменять поля "Тип" и "Название" местами (Тип должен быть сверху). Для каждого типа в выпадающем списке надо сделать отдельную таблицу в БД: Перевозчики, Места доставки, Транспорт, Прицепы, Водители, Склады/Базисы (добавить этот тип в выподающий список). При выборе типа "Склад/Базис" ниже должно появиться поле ввода "Стоимость хранения" для этого склада. Соответсвенно на странице "Справочник логистики" должны выводиться по блокам данные из этих таблиц (не в кучу).
Неиспользуемые поля и таблицы Справочников можно удалить, чтобы не накапливалось пустых полей и легаси таблиц
