Необходимо разработать веб-приложение для компании, работающей на рынке авиатоплива. 
Основные бизнес-процессы:
- Хранение. Выступает как собственная перевалка топлива для дальнейшей поставки клиентам и как услуга. При слиянии нескольких объемов в одном складе выводится усредненная себестоимость.
- Перевозка и доставка. Два разных бизнес-процесса с участием собственного или наемного транспорта. Доставка - часть процесса ОПТовой продажи топлива, когда клиенту требуется поставка топлива на требуемую ему точку слива, в этом случае цена доставки прибавляется к итоговой стоимости продажи. Перевозка - отдельный бизнес-процесс, услуга перевозки топлива, принадлежащего клиенту от базиса (поставщика + его локация) к точке слива.
- ОПТ и Заправка ВС - два основных бизнес-процесса. Опт - поставка топлива путем его передачи покупателю в его автоцистерну из нашей емкости, либо из базиса (самовывоз), а также поставка топлива в требуемую клиенту точку слива с доставкой авто- или жд-транспортом. Заправка ВС представляет из себя 2 процесса: поставку топлива покупателю путем его передачи через пистолет/штуцер в бак ВС и услугу заправки ВС (весь комплекс действий, связанный с заправкой ВС). Они указаны отдельно, т.к. в нашей деятельности есть схемы работы, когда мы продаем только саму услугу, заправляем ВС клиента не своим топливом, другими словами поставки (продажи) топлива в этой схеме нет.

PostgreSQL БД - единый источник правды, и Бэкенд логика с ней связанная:
- ОПТ - таблица учета оптовых продаж. Автоматически заносятся цены и считается стоимость покупки, продажи, доставки (для покупки/продажи нужен контрагент и дата, взятые с UI, для доставки нужно ввести на UI место доставки и перевозчика). Все контрагенты привязаны и выбираются из соответствующего списка на UI (поставщики/покупатели/перевозчики), автоматически указывается базис (для доставки нужно указать на UI место доставки из списка), есть логика проверки действительности и правильности цен, наличия объемов на складах, есть логика для учета транспорта перевозчиков (все известные номера машин и водители уже есть в справочнике).
- Заправка ВС - таблица учета ВСЕХ заправок. Все контрагенты привязаны к спискам, есть автоматизация ввода цен по покупке и продаже за керосин, услугу по заправке, агентские вознаграждения, ПВКЖ, хранение. Автоматически ставится базис поставки, есть логика проверки наличия объемов, действительности и правильности цен
- БИРЖА - таблица для учета биржевых сделок. Содержит всю информацию о каждой биржевой сделке, данные по перемещениям из нее поступают в таблицу "Склады".
- Перемещение - таблица для учета всех поставок топлива и ПВКЖ (противоводнокристаллизационная жидкость) на карты хранения (служба и аэропорты), а также всех внутренних перемещений между нашими складами. Все контрагенты согласно спискам, автоматизация ввода отпускных цен (при внутренних перемещениях автоматически подтягивается себестоимость топлива с карты откуда идет поставка), автоматически ставится цена доставки, автоматически считается себестоимость топлива на месте (карта, куда оно поступило).
- Склады - таблица для учета всех складов (служба, ТК ТВК, аэропорты), а также выделенных месячных объемов ГПН (Газпромнефть) и ГПНА (Газпромнефть Аэро). Содержит ежедневное поступление (автоматически из таблицы "Перемещение", "Биржа"), ежедневный расход (автоматически из таблиц "ОПТ", "Заправка ВС", "Перемещение"), автоматически ежедневно считается себестоимость (при каждом поступлении подсчитывается остаток и поступление и формируется общая цена на весь хранящийся объем), автоматически подтягиваются цены поступившей партии (если их было несколько в течение дня, то считается их общая себестоимость), есть суммы поступления и расхода.
- ЦЕНЫ - таблица для ценообразования. Служит для указания цен покупки и продажи керосина, ПВКЖ, услуг по заправке, хранению, агентских. Все контрагенты и базисы выбираются из соответствующих списков. Каждая цена имеет диапазон действия (даты от/до), дата нужна для автоматического выставления цен в ОПТе и Заправке ВС. Есть возможность поставить несколько различных цен одному контрагенту с одинаковой датой действия.  Есть проверка задвоения цены, есть автоматический подсчет объемов, проданных по указанной цене.
- Стоимость доставки - таблица с перечнем перевозчиков, базисами, местами доставки и тарифами. Из этой таблицы тарифы автоматически подтягиваются в ОПТ и перемещение при расчете доставки.
- Справочники - три вспомогательных таблицы - Все прочие таблицы, где присутствуют ячейки со списками (также выпадающие списки на UI), обращаются к соответствующим таблицам справочников. Содержит три основных таблицы: 1 - Оптовая торговля. Содержит перечень поставщиков, их базисы и перечень покупателей. 2 - Заправки воздушных судов. Содержит перечень аэропортов/поставщиков, их базисы заправки, стоимость услуги по заправке, стоимость ПВКЖ, агентские/прочие услуги, которые данные поставщики оказывают. 3 - Логистика. Содержит перечень мест доставки, перевозчиков, номеров машин, номеров полуприцепов и ФИО водителей перевозчиков, а также склады/базисы и стоимость хранения на них.

UI блоки на фронтенде (роль-ориентированные) + админка:
1. Блок UI для учета оптовых продаж (ОПТ), а также соотвествующие поля БД для таблицы ОПТ:
- Поле выбора даты “Дата” - Указывается дата продажи (должна соответствовать дате товарной накладной).
- Выпадающий список “Поставщик” - Выбираются из списка, расположенного в таблице “Оптовая торговля” (раздела “Справочники”). Нужен для автоматической работы формулы цены закупки.
- Выпадающий список “Покупатель” - Выбираются из списка, расположенного в таблице “Оптовая торговля” (раздела “Справочники”). Нужен для автоматической работы формулы цены продажи.
- Поля ввода “Литры”, “Плотность” или КГ” и переключатель/тумблер, который показыват/скрывает поля Литры, Плотность или поле КГ (может вноситься одно из двух: либо литры и плотность либо КГ) - при этом в поле КГ подставляется результат умножения полей Литры, Плотность.
- Лейбл “Объем на складе” и значение (считается по логике на бэке) - Столбец для проверки наличия требуемого к отгрузке объема на складе Поставщика. Берет информацию из таблицы “Склады”. Если на дату отгрузки остаток на складе ниже суммарного количества отгрузки (суммы всех отгрузок на этот день по данному складу), должно быть написано “нет объема!” (и кол-во недостачи), в противном случае будет стоять “ОК” и значение объема.
- Лейбл “Покупка” и значение (считается по логике на бэке) - Отражает входящую цену. Ставится автоматически, берет данные из таблицы “ЦЕНЫ”, либо из таблицы “Склады”. Для того, чтобы цена встала автоматически, нужно указать дату и поставщика. Если в таблице “ЦЕНЫ” отсутствует цена по поставщику на указанную дату, логика обратится к таблице “Склады”. Если данных по поставщику за требуемую дату нет ни в одной из вышеуказанных таблиц, в ячейке будет написано “нет цены!”, а в суммах закупки/продажи будет стоять ошибка.
- Лейбл “Сумма закупки” и значение (считается по формуле) - Умножает ячейки КГ и Покупка.
- Выпадающий список с ценами - если есть в БД несколько различных цен одному покупателю в течение дня (например, если в один день продали 3 машины покупателю ТЛК с объема ГПН ЯНОС, каждую со своей ценой). В большинестве случаев будет всего одна цена, но для редких случаев нужен выпадающий список цен.
- Лейбл “Продажа” и значение (считается по логике на бэке) - Отражает цену для покупателя. Ставится автоматически, берет данные из таблицы “ЦЕНЫ”. Для того, чтобы цена встала автоматически, нужно указать дату, покупателя и базис (ставится автоматически). Если есть в БД несколько различных цен одному покупателю в течение дня, то берется выбранная в выпадающем списке (см. Пункт выше). Если в таблице “ЦЕНЫ” отсутствует цена по покупателю на указанную дату, в ячейке будет написано “нет цены!”, а в суммах закупки/продажи будет стоять ошибка.
- Выпадающий список “№ договора” - Подставляет из БД договор и приложение, в соответсвии с которыми была создана цена в таблице “ЦЕНЫ”.
- Поле ввода “Примечания” - нужно для указания сопутствующей информации по сделке.
- Лейбл “Базис” и значение (считается по логике на бэке) - Отображает место отгрузки. Устанавливается автоматически по поставщику. Данные берет из таблицы “Справочники”. Для того, чтобы логика автоматически ставила базис, нужно при введении нового поставщика в вышеуказанной таблице поставить наименование базиса справа от названия поставщика.
- Выпадающие списки “Госномер”, “Госномер ПП”, “ФИО водителя” - Выбираются из списка, расположенного в таблице “Логистика” (раздела “Справочники”).
- Выпадающий список “Место доставки” - Данные нужны для корректной работы формулы для поля “Доставка”. Выбираются из списка, расположенного в таблице “Логистика” (раздела “Справочники”).
- Выпадающий список “Перевозчик” - Данные нужны для корректной работы формулы для поля “Доставка”. Выбираются из списка, расположенного в таблице “Логистика” (раздела “Справочники”).
- Лейбл “Доставка” и значение (считается по логике на бэке) - Автоматически считает стоимость доставки. Берет данные из столбцов Базис, Место доставки, Перевозчик, сверяет их с соответствующими столбцами в таблице “Стоимость доставки”. Если все данные совпадают, то берет тариф из этой таблицы, указанный напротив совпавших данных и умножает его на ячейку из столбца КГ.

- Лейбл “Прибыль” и значение (считается по формуле) - Вычитает из суммы продажи сумму закупки и доставку.
- Лейбл “Накопительно” и значение (считается по логике на бэке) - Показывает сумму прибыли текущей и всех предыдущих сделок.
- Элемент Чекбокс (галочка) с наименованием “Примерный объем” - установление признака того, что объем у данной сделки примерный и нужно будет последующее фактическое уточнение (заносится в БД данный признак).
- Область со списком сделок (крайние 10 записей), с фильтрами, пагинацией и возможностью раскрыть на всю страницу, а также контролами управления по нажатию на элемент списка.

- Выпадающий список “Поставщик” - Выбираются из списка, расположенного в таблиц “Оптовая торговля” и “Заправка воздушных судов” (раздел “Справочники”). Нужен для корректной работы формулы цены закупки.
- Лейбл “Базис поставки” и значение (считается по логике на бэке) - Отображает место заправки. Устанавливается автоматически по поставщику. Данные берет из таблицы “Справочники”. Для того, чтобы логика автоматически ставила базис, нужно при введении нового поставщика в вышеуказанной таблице поставить наименование базиса в столбце справа от названия поставщика. Базис нужен для корректной работы формулы цены продажи.

- Лейбл “Цена покупки” и значение (считается по логике на бэке) - Отражает входящую цену. Ставится автоматически, берет данные из таблицы “ЦЕНЫ”, либо из таблицы “Склады”. Для того, чтобы цена встала автоматически, нужно указать дату и поставщика. Если в таблице “ЦЕНЫ” отсутствует цена по поставщику на указанную дату, формула обратится к таблице “Склады”. Если данных по поставщику нет ни в одной из вышеуказанных таблиц, в ячейке будет написано “нет цены!”
- Лейбл “Сумма покупки” и значение (считается по формуле) - Умножает ячейки КГ и Цена покупки.
- Выпадающий список “Перевозчик” - Данные нужны для корректной работы формулы в столбце “Стоимость доставки”. Выбираются из списка, расположенного в таблице “Логистика” (раздел “Справочники”).
- Лейбл “Стоимость доставки” и значение (считается по логике на бэке) - Автоматически считает стоимость доставки. Берет данные из столбцов Базис, Склад, Перевозчик, сверяет их с соответствующими столбцами в таблице “Стоимость доставки”. Если все данные совпадают, то берет тариф из этой таблицы, указанный напротив совпавших данных и умножает его на ячейку из столбца КГ.
- Выпадающий список “Склад” - Указывается склад (базис) поступления товара. Выбирается из списка расположенного в таблице “Логистика” (раздел “Справочники”). Нужен для корректной работы формул по доставке, хранению и складскому учету.
- Лейбл “Хранение” и значение (считается по логике на бэке) - Автоматически считает цену хранения. Берет данные по тарифу за хранение из таблицы “Логистика” (раздел “Справочники”), напротив соответствующего склада и умножает на ячейку из столбца КГ.
- Лейбл “Себестоимость” и значение (считается по формуле) - Автоматически считает себестоимость партии. Складывает закупку, стоимость доставки, хранение и делит эту сумму на КГ.
- Область со списком перемещений (крайние 10 записей), с фильтрами, пагинацией и возможностью раскрыть на всю страницу, а также контролами управления по нажатию на элемент списка. 
- Кнопка Создать - создает новую запись с сохранением занесенных в полях данных в БД.

4. Блок UI для создания/редактирования цены (ЦЕНЫ), а также соотвествующие поля БД для таблицы ЦЕНЫ:
- Поле выбора даты “Дата начала действия” - Указывает с какого числа начинает действовать цена.
- Выпадающий список “Тип” - Выбирает из списка тип контрагента (поставщик/покупатель).
- Выпадающий список “Контрагент” - Выбираются из списка, расположенного в разделе “Справочники” в таблицах “Оптовая торговля” и “Заправка воздушных судов”.
- Выпадающий список “Базис” - Отображает место поставки/заправки. Выбираются из списка, расположенного в разделе “Справочники” в таблицах “Оптовая торговля” и “Заправка воздушных судов”.
- Поле ввода “Цена” - Выставления цен закупки и поставки. Если необходимо назначение нескольких различных цен одному покупателю в течение дня (например, если в один день продали 3 машины покупателю ТЛК с объема ГПН ЯНОС, каждую со своей ценой), то справа есть плюсик для добавления еще одного поля ввода для введение еще одной цены, отличной от первой и т.д., если несколько цен.
- Поле ввода “Объем” - вводится предполагаемый по договору и приложению объем поставки по данной цене.
- Лейбл “Выборка” и значение (считается по логике на бэке) - Автоматически суммирует все сделки из таблиц “ОПТ” и “Заправка ВС”, которые были совершены по этой цене (учитывается только конкретный контрагент, базис и только указанный в данной строке диапазон дат, схожие цены не суммируются).
- Поле выбора даты “Срок действия ДО” - Указывает дату, ДО которой действует цена.
- Выпадающий список “Тип цены” - Указывает тип товара/услуги, на который выставлена цена из списка (керосин/услуга/ПВКЖ/агентские/хранение).
- Индикатор “Проверка дат” (считается по логике на бэке) - Логика автоматической проверки задвоения цен. Если разные цены одному контрагенту пересекаются по своим диапазонам действия (например одна цена действует с 01.05 до 30.05, а вторая действует с 15.05 до 30.05), то все цены, попавшие в задвоенный диапазон (с 15.05 до 30.05) будут просуммированы. То есть если одна цена 32,50, а другая 30,50, то в эти даты формула будет автоматически ставить цену 63,00 в таблицах, куда они попадают. Чтобы избежать подобной ошибки, логика проверяет все диапазоны действия цен по указанному контрагенту на указанном базисе. Если при введении цены был задан диапазон, который пересекается с уже действующим, то формула выдаст запись “двойная цена!” и выведет список всех строках, где найдены пересекающиеся диапазоны.
- Поле ввода “Документы” - вносятся договора/соглашения/прложения в соответсвии с которыми создается/корректируется цена.
- Область со списком цен (крайние 10 записей), с фильтрами, пагинацией и возможностью раскрыть на всю страницу, а также контролами управления по нажатию на элемент списка. 
- Кнопка Создать - создает новую запись с сохранением занесенных в полях данных в БД.
